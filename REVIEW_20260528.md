# 代码审查报告 — 2026-05-28

## 问题 1：高度边界检查与实际最大层数不一致

| 属性 | 值 |
|------|-----|
| **文件** | `scripts/generate_20260528.py` |
| **行号** | 532 |
| **严重度** | 🟡 中 |
| **类型** | 逻辑缺陷（边界条件遗漏） |

### 描述

函数 `generate_20260528()` 的高度上限检查使用了写死的系数 5：

```python
if lobby_height + 5 * floor_height >= 50:
    raise ValueError("building height must be under 50m")
```

而实际的三栋楼层数分别为 `low_block=4`、`mid_block=5`、`high_block=6`。当前的 `5 * floor_height` 仅覆盖到 mid_block（5 层），忽略了 high_block（6 层）可能超出限高的情况。

### 复现条件

当 `floor_height = 9.0` 时：

```
lobby_height(6.0) + 5 × floor_height(9.0) = 51 ≥ 50  → 触发拦截  ✓
low_block 实际高度:  6.0 + (4-1) × 9.0 = 33m          ✓
mid_block 实际高度:  6.0 + (5-1) × 9.0 = 42m          ✓
high_block 实际高度: 6.0 + (6-1) × 9.0 = 51m ≥ 50     ← 被当前检查恰好覆盖
```

当 `floor_height = 8.5` 时：

```
lobby_height(6.0) + 5 × 8.5 = 48.5 < 50  → 通过      ✓
high_block 实际高度: 6.0 + (6-1) × 8.5 = 48.5m       ✓
```

当前写法在 floor_height 为某些值时靠"碰巧"通过了 high_block 的检查，而非精确计算。

### 修复建议

从 `buildings` 列表推导出实际最大层数：

```python
# 替换第 542-545 行的 buildings 定义
buildings = [
    ("low_block", 4, 1.2, low_aspect_ratio, low_offset_angle, low_offset_distance),
    ("mid_block", 5, 1.0, mid_aspect_ratio, mid_offset_angle, mid_offset_distance),
    ("high_block", 6, 0.8, high_aspect_ratio, high_offset_angle, high_offset_distance),
]
# 将第 532 行的检查改为
max_floors = max(b[1] for b in buildings)  # = 6
if lobby_height + (max_floors - 1) * floor_height >= 50:
    raise ValueError("building height must be under 50m")
```

或将校验下移到每个体块生成时单独检查。

## 问题 2：空中平台间碰撞检查缺失

| 属性 | 值 |
|------|-----|
| **文件** | `scripts/generate_20260528.py` |
| **行号** | 405 |
| **严重度** | 🟡 中 |
| **类型** | 校验遗漏 |

### 描述

`validate_collisions()` 对所有 `category == "aerial_platform"` 的 zone 跳过碰撞检测：

```python
def validate_collisions(zones: list[dict]) -> None:
    for index, a in enumerate(zones):
        if a.get("category") == "aerial_platform":   # ← 跳过平台
            continue
        ...
        for b in zones[index + 1:]:
            if b.get("category") == "aerial_platform": # ← 也跳过平台
                continue
```

这意味着：

| 碰撞对 | 是否检测 |
|--------|---------|
| 体块 ↔ 体块 | ✅ |
| 体块 ↔ 平台 | ❌ 跳过 |
| 平台 ↔ 平台 | ❌ 跳过 |

### 当前风险

当前模型有两个平台（`aerial_platform_high_to_low_F03` 和 `aerial_platform_high_to_mid_F04`），分别挂在不同高度（Floor 3 和 Floor 4），Z 轴不重叠，因此不会实际碰撞。但：

- 当 `offset_distance` 较大时，体块的偏移可能导致平台连接路径改变，间接产生 XY 平面重叠
- 后续增加更多平台或自定义楼层时，碰撞风险升高
- 跳过平台与体块的碰撞检测意味着平台可以"穿模"进入体块内部

### 修复建议

```python
def validate_collisions(zones: list[dict]) -> None:
    for index, a in enumerate(zones):
        ax0, ax1, ay0, ay1, az0, az1 = box_bounds(a)
        for b in zones[index + 1:]:
            bx0, bx1, by0, by1, bz0, bz1 = box_bounds(b)
            ox = overlap_1d(ax0, ax1, bx0, bx1)
            oy = overlap_1d(ay0, ay1, by0, by1)
            oz = overlap_1d(az0, az1, bz0, bz1)
            if ox > EPS and oy > EPS and oz > EPS:
                raise ValueError(f"{a['name']} collides with {b['name']}")
```

或保留对平台的跳过，但在 `validate_platform_attachments` 中补充平台间的碰撞检查。

## 问题 3：多边形区域无共享墙面支持

| 属性 | 值 |
|------|-----|
| **文件** | `scripts/idf_converter.py` |
| **引入版本** | Commit `3c09169` / `13ea7cb` |
| **严重度** | 🟡 中 |
| **类型** | 功能缺失（热工模拟精度） |

### 描述

`idf_converter.py` 中的共享墙面检测目前只处理矩形体块（`_ZoneBox`）：

| 区域类型 | 检测函数 | 共享墙面 |
|---------|---------|---------|
| 矩形体块（`_ZoneBox`） | `_detect_horizontal_pairs()` | ✅ 支持拆分 |
| 矩形体块（`_ZoneBox`） | `_detect_vertical_pairs()` | ✅ 支持拆分 |
| 多边形区域（`_ZonePoly`） | 无 | ❌ 所有墙面为 `Outdoors` |
| 多边形区域（`_ZonePoly`） | 无 | ❌ 无垂直堆叠处理 |

`_build_zone_poly()` 生成的墙面全部使用 `Outdoors` 边界：

```python
idf.add(_make_surface(
    name=sname,
    surface_type="Wall",
    construction=ext_wall,       # ← 永远是外墙构造
    zone=zn,
    boundary="Outdoors",         # ← 永远是外墙边界
    sun="SunExposed",
    wind="WindExposed",
    ...
))
```

### 影响

当模型包含多边形 zone（梯形、L 形等 `floor_polygon`）并与其他 zone 相邻时：

- 相邻墙面被 EnergyPlus 视为室外墙 → 增加额外热损失
- 模拟结果的采暖/制冷能耗可能偏高
- 室内热环境模拟不准确

### 复现场景

1. 在 `scripts/generate_*.py` 中使用 `add_polygon_zone()` 创建紧邻的 zone
2. 调用 `convert_and_run()` 生成 IDF
3. EnergyPlus 将相邻面视为外墙，热量流失

**方案 A：多边形退化为包围盒参与墙面检测（短期）**

对多边形 zone 计算其包围盒，用包围盒参与 `_detect_horizontal_pairs` 和 `_detect_vertical_pairs`，然后用包围盒的共享墙面结果近似替换多边形对应的墙面。
